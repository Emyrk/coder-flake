# Workspace, agent, provisioner lifecycle

Tests assert before workspace builds, provisioner jobs, agents, PTYs, log streams, template versions, or app routes reach the expected state.

## Why it flakes

This bucket is core to Coder. It also overlaps with timing, DB-backed lifecycle, and resource flakes, so helper work here pays down several categories at once.

## Common fixes

- Wait for exact lifecycle states: build created, provisioner job started, job complete, logs drained, agent ready, app route available.
- Make timeout errors print the last observed state.
- Use shared helpers instead of ad hoc polling in individual tests.
- Join background goroutines before cleanup cancels contexts.
- Add failure logs around provisioner and agent state transitions.

## Suggested first slice

This is the best first implementation slice: build lifecycle helpers and migrate 3 to 5 existing flaky tests onto them.
